# Doctor-Ready PDF Export — Design Spec

**Issue:** #12
**Date:** 2026-04-12
**Status:** Approved
**Supersedes:** `docs/designs/pdf-export-design.md` (pandoc approach replaced with WeasyPrint)

## Goal

Generate actual PDF reports (not browser print) for doctors, prescribers, and personal reference. Three report types: PGx Card, Mental Health Summary, Full Genome Report.

## Approach

Backend markdown-to-PDF via WeasyPrint (Python). Frontend sends markdown (already generated by existing `*ToMarkdown()` functions) to a new backend endpoint, receives PDF blob, triggers download. No external binaries needed — WeasyPrint is a pip-installable Python package.

## 1. Dependencies

Add to `pyproject.toml` `[web]` extras:
- `weasyprint>=62.0`
- `markdown>=3.5`

Add to `.github/workflows/ci.yml` backend job:
- `apt-get update && apt-get install -y libpango1.0-dev` (WeasyPrint system dependency)

## 2. `pdf_renderer.py` service

**File:** `backend/app/services/pdf_renderer.py`

**Interface:**
```python
def render_pdf(markdown_content: str, report_type: str, metadata: dict | None = None) -> bytes:
    """Convert markdown to PDF bytes via WeasyPrint."""
```

**Flow:**
1. Convert markdown string to HTML via `markdown.markdown()`
2. Wrap HTML in a full document template with embedded CSS
3. Render to PDF bytes via `weasyprint.HTML(string=html).write_pdf()`
4. Return bytes

**CSS template (embedded in HTML):**
- A4, margins 20mm
- Body: EB Garamond 11pt (with fallback to serif)
- Headings: bold, uppercase letter-spacing for section titles
- Tables: border-collapse, alternating gray rows, monospace for gene/variant columns
- `@page` rules: header (report type + date), footer (page number)
- Status indicators: solid/dashed/thin left borders for DANGER/WARN/OK
- Print-safe: black on white, no color dependencies

**Report type differences:**
- `pgx`: title "PHARMACOGENOMICS REPORT", no TOC, disclaimer box
- `mental-health`: title "MENTAL HEALTH GENETIC PROFILE", no TOC
- `full`: title "PERSONAL GENOME REPORT", TOC auto-generated from headings

**Metadata dict:**
```python
{
    "title": str | None,    # override default title
    "date": str | None,     # override today's date
}
```

## 3. Export route

**File:** `backend/app/routes/export.py`

**Endpoint:** `POST /api/export/pdf`

**Request body:**
```json
{
    "markdown": "string (markdown content)",
    "report_type": "pgx | mental-health | full",
    "metadata": { "title": "optional", "date": "optional" }
}
```

**Response:** `application/pdf` with `Content-Disposition: attachment; filename=genome-report-{type}-{date}.pdf`

**Error handling:**
- 400 if markdown is empty or report_type invalid
- 500 if WeasyPrint fails (return error message)

**Registration:** Add `export_router` to `backend/app/main.py`

## 4. Frontend `exportPdf()`

**File:** `frontend/src/lib/export.ts` (add to existing file)

```ts
async function exportPdf(
    markdown: string,
    reportType: 'pgx' | 'mental-health' | 'full',
    metadata?: { title?: string; date?: string }
): Promise<void>
```

**Flow:**
1. POST to `/api/export/pdf` with JSON body
2. Receive blob response
3. Call existing `downloadFile()` with blob content and filename

## 5. UI integration

### PGxPanel.tsx
- Add "EXPORT PDF" button next to existing export buttons
- Calls `exportPdf(pgxToMarkdown(enzymes), 'pgx')`

### MentalHealthDashboard.tsx
- Add "EXPORT PDF" to export options (alongside existing 'md' and 'doctor')
- Calls `exportPdf(mentalHealthToMarkdown(data), 'mental-health')`

### App.tsx — Full Report
- Add export function that concatenates all `*ToMarkdown()` outputs with `---` separators
- Calls `exportPdf(combinedMarkdown, 'full')`
- Triggered from a top-level export button or menu

## 7. Out of scope

- Custom fonts bundled in repo (uses system fonts with fallbacks)
- Patient name/identifier fields
- Digital signatures or watermarks
- Email/share functionality
- Preview before download
